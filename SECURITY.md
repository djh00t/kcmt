# Security Policy

## Supported Versions

Security fixes are applied to the latest tagged release and the current `main` branch only.

| Version | Supported |
| --- | --- |
| Latest tagged release | Yes |
| `main` | Yes |
| Older tags and unmerged branches | No |

## Reporting a Vulnerability

Do not open public GitHub issues for suspected vulnerabilities.

Use GitHub private vulnerability reporting for this repository if it is enabled. If that surface is unavailable, contact the repository owner privately through GitHub before any public disclosure.

Include:

- affected version or commit
- reproduction steps
- impact summary
- any proposed mitigation or workaround

You should receive an acknowledgement within 5 business days. Fix timing depends on severity, exploitability, and whether a safe patch is available.
